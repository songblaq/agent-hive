# AgentHive 로드맵 v1

작성일: 2026-03-09
상태: 초안 v1

## 1. 목표

AgentHive를 파일 기반 멀티 에이전트 협업 프로토콜에서,
실제로 사용할 수 있는 CLI 중심 MVP로 끌어올린다.

## 2. 단계별 계획

### Phase 1. 프로토콜 + 수동 운영

목표:
- 사람이 읽고 에이전트가 따를 수 있는 구조 확정
- 개밥먹기 시작

해야 할 일:
- 아키텍처 문서 작성
- hub 구조 정의
- task 템플릿 정의
- pointer 파일 규칙 정의
- 첫 프로젝트(agent-hive) 등록
- 실제 task 하나 이상 수행

완료 기준:
- 최소 2개 에이전트가 같은 규칙으로 task를 주고받을 수 있음

### Phase 2. CLI MVP

목표:
- 반복 수작업을 명령으로 대체

우선 명령:
- `agenthive init`
- `agenthive project add <path>`
- `agenthive project list`
- `agenthive task create`
- `agenthive task claim <id>`
- `agenthive task complete <id>`
- `agenthive status`
- `agenthive setup <tool>`

완료 기준:
- 허브 초기화와 task lifecycle 핵심 흐름을 CLI로 수행 가능

### Phase 3. Symphony 흡수

목표:
- 상위 orchestration과 연결

해야 할 일:
- Symphony intake → AgentHive task 생성 흐름 정리
- 상태 보고 포맷 통일
- 실행/관제 경계 정의

완료 기준:
- Symphony에서 요청을 받고 AgentHive 상태에 반영할 수 있음

### Phase 4. 자동화

목표:
- 반복 운영의 자율화

후보:
- file watcher
- dead lease 감지
- daily summary
- auto-assign 보조
- kanban 시각화

완료 기준:
- 사람 개입 없이도 일부 루틴 작업이 안정적으로 굴러감

## 3. 우선순위

### 최우선
- repo 구조 정리
- hub 구조 정리
- CLI 명령 범위 확정
- 개밥먹기 task 1회 성공

### 차우선
- pointer file generator
- status 출력 개선
- lock lease 정책

### 후순위
- 웹 UI
- 외부 tracker 연동
- 자동 분배

## 4. 운영 기준

- source of truth는 `~/.agenthive`
- `#openai-symphony`는 intake/control plane
- repo는 공개 코드/문서/템플릿만 담당
- 처음엔 단순하게, 자동화는 나중에

## 5. 성공 기준

아래가 되면 v1 MVP 방향은 맞는 것으로 본다.

- AgentHive가 자기 자신 프로젝트를 등록할 수 있음
- task를 만들고 claim하고 review로 넘길 수 있음
- 최소 2개 이상의 서로 다른 에이전트가 같은 구조를 읽고 협업 가능
- 상태를 사람이 빠르게 파악할 수 있음
