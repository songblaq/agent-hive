# AgentHive 작업계획 v1

작성일: 2026-03-09
상태: 착수용 초안

## 1. 현재 목표

첫 번째 목표는 AgentHive가 자기 자신을 개발하는 개밥먹기 루프를 만드는 것이다.

## 2. 즉시 실행할 작업 묶음

### AH-001 저장소 기본 구조 정리
- 목적: repo를 모든 에이전트가 공통 이해 가능한 형태로 만든다.
- 산출물:
  - README
  - docs/
  - specs/
  - src/
  - templates/

### AH-002 허브 구조 초안 확정
- 목적: `~/.agenthive` 내부 구조를 문서로 먼저 고정한다.
- 산출물:
  - hub 구조 문서
  - registry 예시
  - project/task 예시

### AH-003 CLI 명령 범위 정의
- 목적: MVP에서 구현할 명령과 제외할 명령을 분리한다.
- 산출물:
  - CLI 명령 목록
  - 인자/출력 초안

### AH-004 task 템플릿 정의
- 목적: 사람이든 에이전트든 바로 쓸 수 있는 task card 구조를 만든다.
- 산출물:
  - task.yaml 템플릿
  - plan.md 템플릿
  - summary.md 템플릿
  - message/review/log 샘플

### AH-005 첫 개밥먹기 task 수행
- 목적: 실제로 task를 만들고 claim하고 handoff하는지 검증한다.
- 산출물:
  - 테스트용 첫 task
  - 진행 기록
  - 발견된 문제 목록

## 3. 에이전트 역할 분담 제안

### Claude Code
- 프로토콜/문서 구조화
- task lifecycle 점검
- 규칙의 누락 탐지

### Codex
- TypeScript CLI 골격
- 명령 구조 설계
- 파일 입출력 계층 초안

### OpenCode
- 템플릿/포인터 파일 구조 검토
- 다중 도구 호환성 관점 검토
- 운영 UX 피드백

### BlaqKlawBot(현재 세션)
- 상위 운영 구조 정리
- 초기 문서 통합
- Symphony 연계 관점 정리
- 작업 우선순위와 범위 조정

## 4. 결정이 필요한 항목

아래는 다른 에이전트 결과를 받아 빠르게 확정해야 한다.

- CLI 구현 언어/프레임워크 최종 선택
- lock lease 만료 정책
- scope overlap 검사 규칙
- summary 갱신 권한 규칙
- Symphony와 AgentHive의 자동화 경계

## 5. 완료 정의

아래를 만족하면 착수 단계는 끝난다.

- 모든 에이전트가 같은 목표와 경로를 공유함
- repo에 최소 문서 구조가 존재함
- 첫 번째 task를 생성할 수 있음
- `~/.agenthive`를 실제로 만들 준비가 됨
